## 1. Was ist HTML?

* **Vorheriges Kapitel**: Im vorherigen Kapitel hast du gelernt, wie Browser mit Servern kommunizieren
* Jetzt schauen wir uns an, **was** der Browser dabei eigentlich in der Regel empfängt: **HTML-Dokumente**
* Eine Webseite besteht aus **Inhalten** (Text, Bilder, Videos, Links, etc.), die in einer bestimmten **Struktur** angeordnet sind
* Damit Browser diese Inhalte korrekt verstehen und darstellen können, braucht es eine **Auszeichnungssprache**: **HTML** (HyperText Markup Language)

### HTML ist keine Programmiersprache

:::danger
HTML ist keine Programmiersprache
:::

* **HyperText Markup Language**: HTML steht für HyperText Markup Language.
* **Auszeichnungssprache**: Es ist **keine** Programmiersprache, sondern eine **Auszeichnungssprache**.
* HTML ist übrigens keine Programmiersprache.
* Wenn ihr nur HTML könnt, könnt ihr nicht programmieren."
* HTML ist eine Auszeichnungssprache (nur, falls es noch nicht klar war).

![HTML ist keine Programmiersprache MEME](/img/3-Sprachen%20und%20Methoden%20des%20Internets/html-is-not-a-programming-language.png)

* **Tim Berners-Lee, 1992**: HTML wurde von **Tim Berners-Lee** entwickelt und **1992 veröffentlicht**
    * **Forschungsergebnisse teilen**: Es wurde entwickelt, um **Forschungsergebnisse** auf elektronischem Wege zu teilen und zu verknüpfen
    * **Grundlage des WWW**: Tatsächlich war es die **Grundlage des World Wide Web**
* Mit HTML beschreibt man
    * **Welche Inhalte** auf einer Seite zu sehen sind (Überschriften, Abschätze, Bilder, Links, Texte, usw.)
    * Wie diese **Inhalte logische strukturiert** sind (zB. eine Liste mit Punkten, ein Formular mit Eingabefeldern und einem Absende-Button, usw.)
    * Wie die **Inhalte miteinander verknüpft** sind (zB. ein Link von einer Seite zu einer anderen Seite)
    * **Welche Bedeutung** die Inhalte haben (zB. Überschriften auf verschiedenen Ebenen, Absätze, Hervorhebungen, usw.)
* Mit HTML beschreibt man **nicht**
    * Wie die Inhalte aussehen (also Farben, Schriftarten, Abstände, usw.)
    * Wie die Inhalte sich verhalten (also Animationen, Interaktionen, usw.)

## 2. Wie funktioniert HTML?

* **Textdateien (.html/.htm)**: HTML-Dokumente sind **Textdateien**, die mit der Endung `.html` oder `.htm` gespeichert werden
* **Baumstruktur**: HTML-Dokumente beschreiben eine **Baumstruktur**
* **Elemente verschachtelt**: Diese Baumstruktur besteht aus **Elementen**, die ineinander und nebeneinander verschachtelt sein können
* **Tags (Markierungen)**: Jedes Element wird durch **Tags** (Markierungen) definiert
    * Ein **Start-Tag** markiert den Anfang eines Elements (zB. `<p>` für einen Absatz)
    * Ein **End-Tag** markiert das Ende eines Elements (zB. `</p>` für das Ende eines Absatzes)
    * Zwischen den Tags befindet sich der **Inhalt** des Elements (zB. der Text eines Absatzes)
    * Einige Elemente haben keinen End-Tag und werden als **selbstschließende Tags** bezeichnet (zB. `<img />` für ein Bild)
* Ein HTML-Element besteht aus einem **Start-Tag**, einem **Inhalt** und einem **End-Tag**
* Browser lesen diese Struktur ein und bauen daraus ein **Dokumentenmodell** (**DOM** - Document Object Model) auf

```
html
├── head
│   ├── meta
│   └── title
└── body
    ├── h1
    ├── p
    └── p
        └── a
```

* Das **DOM** wird vom Browser **gerendert** - daraus entsteht die sichtbare Webseite

## 4. Strukturierung von Webseiten (Tags, Elemente, Attribute)

### Verschachtelung von Elementen (Nesting)

* Elemente können andere Elemente **enthalten** (**Verschachtelung** / **Nesting**)
* HTML ist also **hierarchisch**, es bildet sich ein **Baum** (DOM)

<LiveHtml initialHtml={`
<p>Das ist ein <strong>wichtiger</strong> Satz.</p>
`} />

### Block- vs. Inline-Elemente

* **Block-Elemente**: Nehmen die volle Breite ihres Containers ein und beginnen auf einer neuen Zeile (zB. `<div>`, `<p>`, `<h1>`, `<ul>`, usw.)
    * `<div>` (Content Division Element) ist ein generisches Container-Element, das verwendet wird, um andere Elemente zu gruppieren
* **Inline-Elemente**: Nehmen nur so viel Platz ein, wie sie benötigen, und fließen innerhalb des Textes (zB. `<span>`, `<a>`, `<strong>`, usw.)
* Der Unterschied zwischen diesen beiden Arten von Elementen liegt in der Art und Weise, wie sich die Elemente in die Dokumentstruktur einordnen
    * HTML ist zwar nicht für die Darstellung von Inhalten zuständig
    * HTML legt keine Pixel, Farben, Abstände etc. fest.
    * Trotzdem unterscheidet HTML Elemente grob danach, wie sie in den Fluss des Dokuments eingebettet werden.

### Attribute

* **Attribute** liefern zusätzliche Informationen über ein Element
* Attribute werden im **Start-Tag** eines Elements definiert
* Ein Attribut besteht aus einem **Namen** und einem **Wert**, die durch ein Gleichheitszeichen (`=`) getrennt sind
* Der **Attributwert** wird in Anführungszeichen (`"`) gesetzt
* Beispiele
    * **`id`**: Eindeutiger Bezeichner für ein Element innerhalb des Dokuments; kann für Javascript und CSS-Styling verwendet werden
    * **`class`**: Weist einem Element eine oder mehrere Klassen zu, die für CSS-Styling werden können
    * **`href`**: Definiert die Ziel-URL eines Hyperlinks (wird im `<a>`-Element verwendet)
    * **`src`**: Definiert die Quelle eines eingebetteten Mediums (zB. Bild, Video, Audio; wird im `<img>`, `<video>`, `<audio>`-Element verwendet)

## 5. Wichtige HTML-Elemente und Kombinationen

* Einige der folgenden Elemente sind **semantische Elemente**, die nicht nur Struktur, sondern auch **Bedeutung** ausdrücken (mehr dazu in Abschnitt 7)

### Überschriften
* Wichtige Elemente: `<h1>` bis `<h6>`
* **Hierarchie**: `<h1>` ist die höchste Überschrift, `<h6>` die tiefste
* Überschriften helfen, den **Inhalt zu strukturieren** und sind wichtig für die **Barrierefreiheit**
* **Reihenfolge beachten**: Überschriften in der richtigen Reihenfolge verwenden (zB. `<h1>`, dann `<h2>`, dann `<h3>`, usw.)
* **Nicht überspringen**: Vermeide es, Überschriften zu überspringen (zB. von `<h1>` direkt zu `<h3>`) um die Darstellung zu steuern

<h3>Das ist die dritt höchste Überschrift</h3>
`} />

### Absätze

* **`<p>`**: Definiert einen Absatz
* Definieren **Textabschnitte**
* Browser fügen **automatisch Abstände** vor und nach Absätzen ein

<LiveHtml initialHtml={`
<p>Das ist der erste Absatz.</p>
<p>Das ist der zweite Absatz.</p>
<p>Das ist der dritte Absatz.</p>
`} />

### Links und Navigation

* Wichtige Elemente: **`<a>`**, **`<nav>`**
* **Navigationselemente** helfen Benutzern, sich auf der Webseite zurechtzufinden
* **Früher**: `<div>`-Elemente für Navigationen verwendet, das ist für **Screenreader und Suchmaschinen nicht erkennbar**
* **`<nav>`-Element**: Wird verwendet, um einen **Navigationsbereich** zu definieren
* **Listen in Navigation**: Innerhalb des `<nav>`-Elements werden häufig Listen (`<ul>`, `<ol>`, `<li>`) verwendet, um die Links zu strukturieren
* **Nur Hauptnavigation**: `<nav>` sollte nicht um alle Links auf der Seite herum verwendet werden, sondern nur um die Hauptnavigation

<LiveHtml initialHtml={`

    <li><a href="about.html">Über uns</a></li>
    <li><a href="contact.html">Kontakt</a></li>
  </ul>
</nav>
`} />

##### Sonderfälle

###### Interne Sprungmarken

* **`#`-Symbol**: Mit dem `#`-Symbol können **interne Sprungmarken** innerhalb derselben Seite erstellt werden
* **Andere Seite**: Indem dem `#`-Symbol eine Datei oder URL vorangestellt wird, kann auch zu einer bestimmten Stelle auf einer anderen Seite verlinkt werden

<LiveHtml initialHtml={`
<a href="#kapitel2">Weiter zu Kapitel 2</a>
<h2 id="kapitel2">Kapitel 2</h2>
`} />

###### Download-Links

* **`download`-Attribut**: Mit dem `download`-Attribut im `<a>`-Element kann ein Link so gestaltet werden, dass die verlinkte Datei **heruntergeladen** wird, anstatt sie im Browser zu öffnen

<LiveHtml initialHtml={`
<a href="/files/mein-dokument.pdf" download>Dokument herunterladen</a>
`} />

###### E-Mail/Telefon-Links

* **`mailto:`**: Mit dem `mailto:`-Schema können **E-Mail-Links** erstellt werden
* **`tel:`**: Mit dem `tel:`-Schema können **Telefon-Links** erstellt werden
* **Standardprogramm nötig**: Diese Links funktionieren jedoch nur, wenn ein entsprechendes Programm als Standardprogramm hinterlegt ist
* Für `mailto:` ist das in der Regel ein **E-Mail-Client**
* `tel:` funktioniert oft auf **Desktop-Geräten nicht** oder öffnet eine VoIP-Anwendung

<LiveHtml initialHtml={`

    <a href="tel:+49123456789">Anrufen</a>
`} />

* **`<img>`**: Fügt Bilder in die Webseite ein
* **`src`-Attribut**: Gibt die **Bild-URL** an (Source)
* **`alt`-Attribut**: Beschreibt das Bild für **Screenreader** und wird angezeigt, wenn das Bild nicht geladen werden kann (**Alternative Text**)

* **`<ul>`**: **Ungeordnete Liste** (Punkte)
* **`<ol>`**: **Geordnete Liste** (Nummern)
* **`<li>`**: **Listenelement**

* **`<table>`**: Definiert eine **Tabelle**
* **`<tr>`**: Definiert eine **Tabellenzeile** (Table Row)
* **`<td>`**: Definiert eine **Tabellenzelle** (Table Data - Datenzelle)
* **`<th>`**: Definiert eine **Kopfzelle** (Table Header)

### Formulare

* Wichtige Elemente: `<form>`, `<input>`, `<textarea>`, `<button>`, `<select>`, `<option>`
* **Interaktive Elemente**: Formulare sind interaktive Elemente, die Benutzern ermöglichen, Daten einzugeben und an den Server zu senden
* **`<form>` umschließt alles**: Das `<form>`-Element umschließt alle Formularfelder und definiert die Aktion und Methode für die Datenübertragung
* Wichtige Attribute:
    * **`action`**: Die URL, an die die Formulardaten gesendet werden
    * **`method`**: Die HTTP-Methode für die Datenübertragung (`get` oder `post`)
    * → Details zu HTTP-Methoden folgen in Kapitel 4 (Backend-Entwicklung)
* Innerhalb des `<form>`-Elements werden verschiedene Eingabeelemente verwendet, um unterschiedliche Arten von Daten zu erfassen
* `<input type="text">`: Definiert ein Eingabefeld (zB. Textfeld, Passwortfeld, usw.)
    * Verschiedene `type`-Attribute sind möglich (zB. `text`, `password`, `email`, `checkbox`, `radio`, usw.)
* `<textarea>`: Definiert ein mehrzeiliges Texteingabefeld
* `<button type="submit">`: Definiert einen Button
    * Das `type`-Attribut kann `submit`, `reset` oder `button` sein
        * `submit` sendet das Formular ab
        * `reset` setzt das Formular zurück
        * `button` ist ein allgemeiner Button, an den über JavaScript Funktionen gebunden werden können
* `<select>`: Definiert eine Dropdown-Liste
* `<option value="foo">`: Definiert eine Option in einer Dropdown-Liste
    * Das `value`-Attribut gibt den Wert der Option an, der beim Absenden des Formulars gesendet wird
    * Der Text zwischen den `<option>`-Tags ist der sichtbare Text der Option

            </form>
`} />

## 6. Der head-Bereich & Metadaten

* **Metadaten & Informationen**: Der `<head>`-Bereich eines HTML-Dokuments enthält Metadaten und andere Informationen
    * **Nicht sichtbar**: Die Inhalte des `<head>`-Bereichs werden nicht direkt auf der Webseite angezeigt
    * **Innerhalb `<html>`, außerhalb `<body>`**: Der `<head>`-Bereich befindet sich innerhalb des `<html>`-Elements, aber außerhalb des `<body>`-Elements
    * Der `<head>`-Bereich enthält **Metadaten** über die Seite:
        * **Browser-Informationen** (Zeichensatz, Titel)
        * **Suchmaschinen-Informationen** (Beschreibung, Schlüsselwörter)
        * **Externe Ressourcen** (CSS-Dateien, JavaScript-Dateien)
        * **Social Media Tags** (Open Graph für Link-Vorschauen)

### Wichtige Elemente im head-Bereich

* `<meta charset="UTF-8">`: Definiert den Zeichensatz des Dokuments (UTF-8 ist der Standard und wird empfohlen)
* `<title>`: Definiert den Titel der Webseite, der im Browser-Tab angezeigt wird
* `<meta name="description" content="Beschreibung der Webseite">`: Definiert eine kurze Beschreibung der Webseite, die von Suchmaschinen verwendet wird
* `<meta name="keywords" content="Schlüsselwort1, Schlüsselwort2, Schlüsselwort3">`: Definiert Schlüsselwörter für die Webseite (wird von Suchmaschinen verwendet)
* `<link rel="stylesheet" href="styles.css">`: Verweist auf eine externe CSS-Datei für das Styling der Webseite
* `<script src="script.js" defer></script>`: Verweist auf eine externe JavaScript-Datei (das `defer`-Attribut sorgt dafür, dass das Skript erst nach dem Laden des HTML-Dokuments ausgeführt wird)
* `<link rel="icon" href="favicon.ico" type="image/x-icon">`: Definiert das Favicon der Webseite (das kleine Symbol, das im Browser-Tab angezeigt wird)
* `<meta property="og:title" content="Titel für soziale Medien">`: Open Graph Tag für den Titel, der angezeigt wird, wenn die Seite in sozialen Medien geteilt wird
* `<meta name="viewport" content="width=device-width, initial-scale=1.0">`: Sorgt dafür, dass die Webseite auf mobilen Geräten korrekt skaliert wird (wichtig für responsives Design)

## 7. Barrierefreies HTML

### Was ist Barrierefreiheit und warum ist sie wichtig?

* **Accessibility (a11y)**: Barrierefreiheit (Accessibility, kurz a11y) bedeutet, dass Webseiten und Anwendungen so gestaltet sind, dass sie von möglichst vielen Menschen genutzt werden können, unabhängig von ihren Fähigkeiten oder Einschränkungen
* **Zugang für alle**: Barrierefreiheit ist wichtig, weil sie sicherstellt, dass alle Menschen Zugang zu Informationen und Dienstleistungen im Internet haben
* **Gesetzlich vorgeschrieben**: Barrierefreiheit ist auch gesetzlich vorgeschrieben (zB. in der EU durch die Web Accessibility Directive)
* **Nutzen für alle**: Barrierefreiheit verbessert die Benutzerfreundlichkeit für alle Nutzer, nicht nur für Menschen mit Behinderungen
    * Beispiel: Gute Kontraste, klare Strukturen und einfache Navigation kommen allen Nutzern zugute
* **Bessere SEO**: Barrierefreiheit kann auch die Suchmaschinenoptimierung (SEO) verbessern, da Suchmaschinen den Inhalt und die Struktur der Webseite besser verstehen können
* **Kontinuierlicher Prozess**: Barrierefreiheit ist ein kontinuierlicher Prozess und sollte von Anfang an in den Entwicklungsprozess integriert werden
* **WCAG-Standards**: Es gibt verschiedene Richtlinien und Standards für barrierefreies Webdesign, zB. die Web Content Accessibility Guidelines (WCAG)
* Barrierefreiheit umfasst verschiedene Aspekte:
    * **Textalternativen** für Bilder (Alt-Texte)
    * **Tastatur-Navigation** ohne Maus
    * **Screenreader-Kompatibilität**
    * **Farbkontraste** für bessere Lesbarkeit
    * **Konsistente Navigation**
    * **Verständliche Sprache**
* Ein paar dieser Aspekte lassen sich durch einfache Maßnahmen in Form von HTML umsetzen
* Weitere Aspekte betreffen das Design (CSS) oder die Funktionalität (JavaScript) und werden in späteren Kapiteln behandelt
* Schließlich lassen sich manche Aspekte nur durch den Inhalt selbst lösen (zB. verständliche Sprache)

### Semantic HTML

* **Semantik = Bedeutung**
* **Bedeutung des Inhalts**: In HTML bedeutet das: Elemente beschreiben nicht nur die Struktur, sondern die **Bedeutung** des Inhalts
* Einfaches Beispiel:
    * `<b>` vs. `<strong>`
        * **`<b>`**: Macht den Text fett, hat aber keine besondere Bedeutung
        * **`<strong>`**: Macht den Text fett und signalisiert, dass der Text wichtig ist
* **Vorteile**:
    * **Bessere Zugänglichkeit**: Screenreader können die Bedeutung besser interpretieren
    * **SEO**: Google erkennt besser, welche Inhalte wichtiger sind
    * **Bessere Wartbarkeit**: Andere Entwickler verstehen die Struktur und Bedeutung besser
* **Nicht für Optik**: Auch für semantic HTML gilt: HTML ist nicht für die Optik zuständig
    * Semantic HTML beschreibt die Bedeutung und Struktur des Inhalts
    * Wie die Inhalte aussehen, wird durch CSS definiert

#### Wichtige semantische HTML-Elemente in HTML5

:::info
HTML5 ist die 5. Hauptversion von HTML und wurde 2014 veröffentlicht.

Im Folgenden werden einige der wichtigsten semantischen HTML-Elemente vorgestellt, die in HTML5 eingeführt wurden. Die Liste ist nicht vollständig, zeigt aber die Funktionsweise und den Zweck dieser Elemente.

**Tipp**: Du musst nicht alle Elemente auswendig kennen - wichtig ist das Verständnis des Konzepts und der Zweck semantischer Elemente.
:::


##### Strukturierende Elemente

* `<header>`: Definiert den Kopfbereich einer Seite oder eines Abschnitts (zB. Logo, Navigation, Überschrift)
* `<nav>`: Definiert einen Navigationsbereich mit Links zu anderen Seiten oder Abschnitten
* `<main>`: Definiert den Hauptinhalt der Seite (sollte nur einmal pro Seite verwendet werden)
* `<section>`: Definiert einen thematisch zusammenhängenden Abschnitt (zB. Kapitel, Artikel, usw.)
* `<article>`: Definiert einen eigenständigen Inhalt (zB. Blog-Post, Nachricht)
* `<aside>`: Definiert Inhalte, die indirekt mit dem Hauptinhalt verbunden sind (zB. Seitenleiste, Werbung)
* `<footer>`: Definiert den Fußbereich einer Seite oder eines Abschnitts (zB. Kontaktinformationen, Copyright-Hinweise)

##### Textbezogene Elemente

* **`<em>`**: Betont Text (meist kursiv dargestellt)
* **`<strong>`**: Wichtiger Text (meist fett dargestellt)
* **`<blockquote>`**: Längere Zitate
* **`<code>`**: Quellcode oder Programmierbeispiele
* **`<abbr>`**: Abkürzungen (mit `title`-Attribut erklärbar)

## Zusammenfassung

✅ **HTML ist eine Auszeichnungssprache** (keine Programmiersprache!) zur Strukturierung von Webinhalten

✅ **Tags definieren Elemente**, die verschachtelt werden können und eine Baumstruktur (DOM) bilden

✅ **Semantisches HTML** beschreibt die Bedeutung des Inhalts und verbessert Barrierefreiheit und SEO
